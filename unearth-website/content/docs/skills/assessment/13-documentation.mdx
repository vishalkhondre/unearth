## What This Skill Does

Compiles all upstream pipeline outputs into a single, comprehensive architecture document. Rather than repeating raw findings, this skill synthesizes, summarizes, and cross-references them into a navigable deliverable that a technical or business stakeholder can read end-to-end or use as a reference.

Without this step, the pipeline produces 12 separate output files that each answer a different question. This skill weaves them into one document that tells the complete story: what the system is, what it does, how it is built, what is working, what is broken, and what to do next.

## Process Steps

1. **Determine Document Scope.** Check which upstream outputs are available. With a minimum of 8 skills, produce sections 1-6 only. With all 13, produce the full document including assessment sections.
2. **Write the Executive Summary.** Produce a 1-page summary covering system name, technology stack, scale, architecture style, key strengths, key risks, blueprint coverage, and recommended next steps.
3. **Write the Architecture Overview.** Compile system context (actors, external systems, boundary), container architecture (inventory, topology, key flows), and component architecture (layer structure, component inventories).
4. **Write the Capability Map.** Synthesize L1/L2 capabilities, the capability-to-actor matrix, and capability-to-container alignment.
5. **Write the Data Model Summary.** Summarize entity inventory, key entity profiles, relationship summary, and coupling hotspots.
6. **Write the Assessment Sections.** Synthesize pattern assessment, dependency landscape, blueprint comparison scorecard, tech debt top findings, and migration readiness.
7. **Write Recommendations and Open Questions.** Merge roadmaps from S-10 and S-11 into a unified list by time horizon. Compile all `[NEEDS VERIFICATION]` items from every upstream output into a single register.
8. **Compile Final Document.** Assemble with table of contents and section cross-references.

## Key Output Sections

| Section | Content |
|---------|---------|
| Executive Summary | 1-page overview: system identity, scale, strengths, risks, and top 3 recommended actions |
| Architecture Overview | System context, container architecture, and component architecture from S-00 through S-05 |
| Capability Map | L1/L2 capabilities with actor matrix and container alignment from S-06 |
| Data Model Summary | Entity inventory, key entities, relationships, and coupling hotspots from S-07 |
| Pattern Assessment | Detected patterns, anti-patterns, and absent patterns from S-08 |
| Dependency Landscape | Dependency statistics, coupling points, and EOL risks from S-09 |
| Blueprint Comparison | Coverage scorecard, critical/moderate gaps, and overlap summary from S-10 |
| Tech Debt Summary | Top debt items by severity, category breakdown, and quick wins from S-11 |
| Migration Readiness | Component extraction difficulty and key blockers from S-12 |
| Recommendations | Unified near-term, mid-term, and long-term roadmap merged from S-10 and S-11 |
| Open Questions | All `[NEEDS VERIFICATION]` items from all upstream outputs, grouped by theme |

## Verification Gate

Before marking complete, the output must pass:

- Executive summary fits on 1 page with all 8 required elements
- Architecture overview covers context, containers, and components from S-00 through S-05
- Capability map includes L1/L2 list and capability-to-actor matrix from S-06
- Assessment sections only include content from skills that were actually run
- Recommendations are traceable to specific findings (G-ID, TD-ID, R-ID, P-ID)
- Open questions compile all `[NEEDS VERIFICATION]` items from all upstream outputs
- No new findings are introduced -- this skill synthesizes only
- All uncertain findings are flagged as `[NEEDS VERIFICATION]`
