## Skill Anatomy

Every skill follows the same structure:

1. **YAML Frontmatter** — name and description for tool integration
2. **Pipeline Position** — stage, order, upstream/downstream dependencies
3. **Inputs** — required and optional data from previous skills
4. **Process Steps** — numbered steps the agent follows
5. **Checkpoints** — intermediate verification pauses
6. **Rationalizations to Resist** — common shortcuts the agent should avoid
7. **Red Flags** — failure modes to watch for
8. **Output Specification** — exact format and sections of the output file
9. **Verification Gate** — checklist the output must pass
10. **Stack Adapter Hooks** — where framework-specific hints plug in
