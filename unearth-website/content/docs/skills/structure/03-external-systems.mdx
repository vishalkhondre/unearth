## What This Skill Does

Maps every external dependency the system communicates with by scanning package references, connection strings, configuration files, and HTTP service client patterns. Each external system is classified by type, access direction, and which projects use it.

Without this step, the C4 Context diagram will be incomplete (missing external systems the platform depends on), and the Container diagram will lack the data stores, message brokers, and identity providers that containers connect to.

## Process Steps

1. **Load upstream context** -- Read the tech stack and project list from `00-recon`, the boundary definition from `02-actors-and-boundaries`, and optionally non-HTTP trigger sources from `01-entry-points`
2. **Scan package references** -- Search dependency manifests (csproj, package.json, requirements.txt, pom.xml, etc.) for packages that map to external systems: database drivers, message broker clients, cloud SDKs, identity providers, observability, and API clients
3. **Scan configuration files** -- Search for connection strings, service endpoint URLs, API keys, queue/topic names, and storage account references in framework-specific config files
4. **Scan for HTTP service clients** -- Find external API calls via HttpClient, IHttpClientFactory, Refit, RestSharp, axios, requests, gRPC channels, and SOAP/WCF proxies
5. **Classify external systems** -- Categorize each as data store, cache, messaging, identity and security, API gateway, observability, compute, analytics, or adjacent service, and record access direction (read, write, or read+write)
6. **Cross-reference with boundary definition** -- Compare findings against the "Outside the Boundary" table from `02-actors-and-boundaries` and flag discrepancies or new discoveries
7. **Build package-to-system mapping** -- Create a reusable reference table mapping every relevant package to its external system
8. **Compile external systems output** -- Assemble findings into the output file

## Verification Gate

Before marking complete, the output must pass:

- Every project from `outputs/00-recon.md` Section 4 has been scanned for package references
- Every relevant package is mapped to an external system
- Configuration files have been scanned for connection strings and service endpoints
- HTTP service clients have been identified and mapped to external systems
- Every external system has ID, purpose, classification, packages, projects, config keys, access direction, and evidence
- Cross-reference with the boundary definition from `02-actors-and-boundaries` is done and discrepancies flagged
- All uncertain findings are flagged as `[NEEDS VERIFICATION]`
