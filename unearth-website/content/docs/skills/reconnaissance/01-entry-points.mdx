## What This Skill Does

Discovers every way into the system -- every HTTP route, event trigger, scheduled job, message handler, WebSocket hub, and background worker. The entry point inventory defines the system's surface area and is the foundation for identifying actors, drawing boundaries, and mapping capabilities.

Without this step, the agent will miss non-obvious entry points (event triggers, blob triggers, change feeds, background workers) that often contain critical business logic invisible from the API surface alone.

## Process Steps

1. **Load recon context** -- Read the project list and technologies from `outputs/00-recon.md` to know which directories to scan
2. **Scan for HTTP entry points** -- Find all REST/API endpoints using framework-specific patterns (controller attributes, route decorators, minimal API mappings)
3. **Scan for event and message triggers** -- Search for message queue handlers, event stream consumers, change feed processors, blob triggers, and webhook receivers
4. **Scan for scheduled jobs and background workers** -- Find timer/cron triggers, hosted services, background workers, and startup/shutdown hooks
5. **Scan for WebSocket and real-time endpoints** -- Detect SignalR hubs, Socket.io handlers, gRPC services, GraphQL endpoints, and raw WebSocket handlers
6. **Scan for CLI and console entry points** -- Find command-line interfaces, management commands, and console applications
7. **Classify entry points by auth level** -- Tag each entry point as anonymous, API key, JWT bearer, role-based, internal/system, or custom
8. **Compile summary statistics** -- Count and reconcile all entry points by category, flag notable patterns (zero entry point projects, missing health endpoints, inconsistent auth)
9. **Compile entry points output** -- Assemble findings into the output file

## Verification Gate

Before marking complete, the output must pass:

- Every project from `outputs/00-recon.md` Section 4 is scanned for entry points
- Projects with zero entry points are explicitly noted with the reason (library, tooling, SPA)
- All HTTP endpoints are recorded with file path, method, route, and auth level
- All non-HTTP triggers are recorded with file path, trigger type, source, and connection config
- Scheduled jobs and background workers are recorded (or explicitly absent with significance noted)
- Auth levels are classified for all endpoints
- Summary table totals are reconcilable (categories sum to total)
- All uncertain findings are flagged as `[NEEDS VERIFICATION]`
