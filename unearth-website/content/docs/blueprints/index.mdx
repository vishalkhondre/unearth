A **blueprint** is a reference capability model for a specific industry domain. Use it with skill `10-blueprint` to assess how your system compares to industry expectations.

## How Blueprints Work

During skill `10-blueprint`, the agent:

1. Maps each discovered L1 capability to the closest blueprint capability
2. Rates coverage as **Full**, **Partial**, **Gap**, or **N/A**
3. Documents what exists versus what is missing for partial matches
4. Classifies gaps as **Critical** (blocks core operations) or **Moderate** (limits growth)
5. Produces a prioritized roadmap for closing gaps

## Creating Custom Blueprints

If your domain is not covered, you can create a custom blueprint. See the [Creating Blueprints](/docs/advanced/creating-blueprints) guide.

A good blueprint has 15--35 capabilities, is grounded in real industry frameworks (not invented), and includes evidence signals that help the agent match capabilities to code patterns.
